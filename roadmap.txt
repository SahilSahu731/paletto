# Color Palette Generator - Complete Roadmap

## **Phase 1: Foundation & Core Setup** (Days 1-2)

### Checkpoint 1.1: Project Structure
1.1.1. Initialize Next.js project with TypeScript, Tailwind, and shadcn/ui
1.1.2. Set up Express.js backend with proper folder structure (routes, controllers, models, middleware)
1.1.3. Configure MongoDB connection with Mongoose
1.1.4. Set up environment variables for both frontend and backend
1.1.5. Configure CORS for frontend-backend communication
1.1.6. Set up basic error handling middleware on backend

### Checkpoint 1.2: Authentication System
1.2.1. Design User schema in MongoDB (email, password hash, created date)
1.2.2. Build registration endpoint with password hashing
1.2.3. Build login endpoint with JWT token generation
1.2.4. Create protected route middleware for Express
1.2.5. Set up Zustand store for auth state management on frontend
1.2.6. Build login and signup UI components
1.2.7. Implement token storage and automatic auth checking on frontend
1.2.8. Add logout functionality

**Phase 1 Complete When:** User can register, login, logout, and authentication persists on page refresh

## **Phase 2: Image Upload & Color Extraction** (Days 3-5)

### Checkpoint 2.1: Image Upload System
2.1.1. Design file upload UI with drag-and-drop zone
2.1.2. Add file type validation (only images: jpg, png, webp)
2.1.3. Add file size validation (max 5MB)
2.1.4. Implement image preview after upload
2.1.5. Set up image upload endpoint on backend
2.1.6. Configure file storage strategy 
2.1.7. Handle image compression if needed

### Checkpoint 2.2: Color Extraction Algorithm
2.2.1. Research and choose color extraction library (node-vibrant, color-thief, or sharp with custom logic)
2.2.2. Install necessary image processing libraries
2.2.3. Build color extraction endpoint that accepts image
2.2.4. Implement dominant color extraction (5-10 main colors)
2.2.5. Calculate color frequencies/prominence
2.2.6. Return colors in multiple formats (hex, rgb, hsl)
2.2.7. Test with various image types and sizes
2.2.8. Add loading states on frontend during extraction

### Checkpoint 2.3: Display Extracted Colors
2.3.1. Design color palette display UI (color swatches)
2.3.2. Show color codes (hex, rgb, hsl) for each color
2.3.3. Add copy-to-clipboard functionality for each color code
2.3.4. Display color name approximations
2.3.5. Show color percentages/prominence in image
2.3.6. Add hover effects and interactions

**Phase 2 Complete When:** User can upload image and see extracted colors with all their codes, can copy any color code

## **Phase 3: Generate Palettes from Single Color** (Days 6-7)

### Checkpoint 3.1: Color Input System
3.1.1. Build color picker UI component
3.1.2. Add manual hex code input field with validation
3.1.3. Allow clicking on extracted colors to use as base
3.1.4. Implement color preview for selected color

### Checkpoint 3.2: Palette Generation Algorithms
3.2.1. Research color harmony rules (complementary, analogous, triadic, tetradic, monochromatic)
3.2.2. Implement complementary color generation
3.2.3. Implement analogous color generation (colors next to each other on wheel)
3.2.4. Implement triadic color generation (3 evenly spaced colors)
3.2.5. Implement tetradic/square color generation (4 evenly spaced)
3.2.6. Implement monochromatic generation (shades and tints of one color)
3.2.7. Build backend endpoint for each generation type
3.2.8. Add algorithm selection UI on frontend

### Checkpoint 3.3: Generated Palette Display
3.3.1. Display generated palette with all variations
3.3.2. Show harmony type name (e.g., "Complementary", "Triadic")
3.3.3. Allow switching between different harmony types
3.3.4. Show all color codes for generated colors
3.3.5. Add copy functionality for generated colors

**Phase 3 Complete When:** User can input any color and generate multiple palette types, copy all generated colors
